# 偶像大师闪耀色彩 - 卡牌培育机制说明

## 项目定位
本项目是闪耀色彩×学园偶像大师的同人联动作品，使用闪耀色彩的角色数据，结合学园偶像大师的卡牌培育玩法。

---

## 一、卡片系统

### 1.1 育成卡（Produce卡，P卡）

育成卡是培育的核心，每张育成卡代表一个特定主题的偶像形象。

**稀有度**：R、SR、SSR、UR（从低到高）

**卡片属性**（随机分配）：
- 三维基础值：Vocal（歌唱）、Dance（舞蹈）、Visual（形象）
- 体力上限
- 培育计划：感性（センス）、理性（ロジック）、非凡（アノマリー）
- 专属技能卡（由AI生成，与角色主题相关）

**UR卡特殊规则**：
- 【絵空靴】杜野凛世 → 感性
- 【誘爆ハートビート】黛冬優子 → 非凡
- 【アマテラス】樋口円香 → 理性
- 其他UR卡根据角色特性指定培育计划

**卡片强化**：
- 使用星尘（抽卡获得的货币）进行界限突破
- 提升三维基础值、体力上限、专属技能卡效果

**不包含的系统**（相比学园）：
- ❌ 专属P道具（取消）
- ❌ 技能书和金币特训（取消）
- ❌ P卡碎片突破（改用星尘）

### 1.2 支援卡（Support卡，S卡）

⚠️ **暂不开发** - 等所有项目完成后，根据时间决定是否实现。

## 二、培育计划与效果体系

### 2.1 三大培育计划

每张P卡属于以下其中一种培育计划，决定了其技能卡、buff、玩法风格：

#### 感性（センス，Sense）

**核心效果**：
- **好调（状态良好）**：技能卡的得分量增加50%，层数在每回合开始时-1
- **集中**：每层集中增加N点技能卡得分
- **绝好调（状态绝佳）**：使"状态良好"的得分增加倍率额外提升（N×10）%

**玩法特点**：通过堆叠"状态良好"和"集中"层数，爆发式地提高得分。

#### 理性（ロジック，Logic）

**核心效果**：
- **好印象**：每层好印象在回合结束时直接获得N点得分，层数在每回合开始时-1
- **有干劲（やる気）**：每层有干劲在打出提供元气的技能卡时，额外获得N点元气

**玩法特点**：通过稳定积累"好印象"和"有干劲"，持续获得分数和元气资源。

#### 非凡（アノマリー，Extraordinary）

**核心机制**：三种指针状态（全力、坚决、温存）

- **全力**：
  - 积累全力值（最大10点），全力值满后可转化为全力状态
  - 全力状态下：分数提升量增加200%，技能卡使用次数+1，将保留的技能卡移动至手牌
  - 全力状态结束前不可转化为坚决或温存

- **坚决**：
  - 第一阶段：分数提升量和体力消耗均增加100%
  - 第二阶段：分数提升量增加150%，体力消耗增加100%，且每使用一张技能卡额外消耗1点体力

- **温存**：
  - 第一阶段：分数提升量和体力消耗均减少50%，解除时热情+5，技能卡使用次数+1
  - 第二阶段：分数提升量和体力消耗均减少75%，解除时热情+8，元气+5，技能卡使用次数+1

**玩法特点**：通过灵活切换三种状态，适应不同的战术需求（爆发/稳健/蓄力）。

## 三、技能卡系统

### 3.1 技能卡基础

技能卡是培育中最核心的元素，通过打出技能卡获取得分和效果，完成训练和比赛。

**稀有度**：N、R、SR、SSR（N卡包括"基本"技能卡和陷阱卡，除陷阱卡外无法额外获取）

**卡片类型**：
- **A（アクティブ，Active，行动）**：六边形外框，可直接获取得分的技能卡
- **M（メンタル，Mental，精神）**：圆形外框，不直接获取得分，而仅提供加成效果的技能卡
- **T（トラブル，Trap，陷阱）**：方形外框，仅提供负面效果的技能卡（如"眠気"）

**卡片效果示例**（基于学园偶像大师）：
- **属性操作**：元气+10、干劲+4、专注+3、消耗全力值3点
- **Buff效果**：状态良好+2回合、好调+3回合、集中+2层、体力消耗减少+2回合
- **数值增幅**：数值上升量增加20%、好印象增加为1.1倍、得分+X（基于当前属性）
- **特殊效果**：技能卡使用数+1、下一张卡效果发动2次、将3张SSR移至牌堆顶
- **消耗/回复体力**：消耗5体力、体力回复+2

**注意**：技能卡不直接提升Vocal/Dance/Visual三维，而是通过属性和Buff间接影响训练/比赛得分

**强化机制**：
- 技能卡可以进行一次强化，提升其效果或降低消耗
- 已强化的技能卡会显示"+"
- 陷阱卡不提供任何正面效果，故无法强化

### 3.2 技能卡的使用

**消耗**：
- 优先消耗元气，元气不足时消耗体力
- 带有"体力消费"（红色图标）效果的技能卡无法消耗元气，只能直接消耗体力

**手牌机制**：
- 每回合从"牌组"（山札）中随机抽取3张技能卡作为"手牌"（手札）
- 每回合最多同时拥有5张手牌
- 默认情况下每回合只能使用1张技能卡（可通过道具、饮料、技能卡效果增加）

**弃牌与重组**：
- 使用后的技能卡进入"弃牌堆"（捨札）
- 部分技能卡单次训练仅限使用一次，使用后进入"除外堆"
- 若"牌组"中技能卡数量不足，则先抽出剩余卡牌，再将"弃牌堆"洗牌组成新牌组后继续抽取

**跳过出牌**：
- 在还拥有出牌次数时，可以选择跳过出牌回合以回复2点体力（不可超过体力上限）

### 3.3 专属技能卡（AI生成）

每张P卡都有自己的专属技能卡，这些技能卡由AI根据以下规则生成：

**生成规则**：
- 根据角色主题、稀有度、培育计划、属性，生成符合角色特色的技能卡
- 使用"Card Generation Bot"（专用AI角色卡）生成
- 生成后的技能卡会永久绑定到该P卡上

**重要说明**：
- ❌ 技能卡**不直接提升Vocal/Dance/Visual三维**
- ✅ 技能卡的效果主要包括：**属性操作**（元气、干劲、专注等）、**Buff效果**（状态良好、好调等）、**特殊效果**（使用数+1、数值增幅等）
- ✅ Vocal/Dance/Visual三维的作用：
  1. 设置**审查基准**（决定比赛中各属性回合的分布）
  2. 在比赛/训练中提供**分数乘区**（三维越高，在对应主题回合内，技能卡获得的分数越高）

**效果类型示例**（基于学园偶像大师）：

| 效果类型 | 说明 | 示例 |
| ------ | ---- | ---- |
| **属性操作** | 获得/消耗：元气、干劲、精力、专注、集中、活力、全力值等 | 元气+10、消耗专注3、干劲+4 |
| **Buff获得** | 获得持续性状态效果 | 状态良好+2回合、好调+3回合、集中+2层 |

| **特殊效果** | 改变游戏规则的效果 | 技能卡使用数+1、将3张SSR移至牌堆顶、下回合手牌+2 |
| **消耗体力** | 需要消耗体力才能使用 | 消耗5体力、体力消耗减少+2回合 |

**非凡系统专属技能卡**（特殊机制）：

非凡培育计划拥有独特的技能卡机制，涉及**全力值**、**温存状态**、**坚决状态**、**热意值**等特殊系统：

| 卡牌名称 | Cost | 效果 | 特点 |
| ------ | ---- | ---- | ---- |

**非凡系统关键词**：
- **全力状态**：非凡独有的特殊状态，可触发强力效果

- **坚决状态**：第三种状态
- **热意值**：非凡独有的属性，类似感性的"好印象"或理性的"集中"

**AI生成技能卡的提示词要求**：

生成技能卡时，AI必须遵循以下规则：
1. **绝对不能**直接提升Vocal/Dance/Visual三维
2. **优先使用**：元气、干劲、精力、专注、集中、活力、全力值等属性
3. **合理搭配**：状态良好、好调、状态绝佳、好印象、体力消耗减少等Buff
4. **创意效果**：技能卡使用数+1、下一张卡效果发动2次、数值上升量增加X%、移动特定卡牌等
5. **平衡性**：根据稀有度控制效果强度和复杂度
   - **R卡**：极简单、单一效果（如"好印象+6"、"元气+10"）
   - **SR卡**：2-3个效果组合，或带条件的强力效果
   - **SSR卡**：多个强力效果组合，或消耗Buff触发超强效果
   - **UR卡**：改变规则的机制，或SSR效果的2倍强度
6. **培育计划关联**：
   - **感性（Sense）**：偏向情感类效果（好印象、好调、活力、状态良好）
   - **理性（Logic）**：偏向逻辑类效果（集中、专注、数值增幅、干劲）
   - **非凡（Anomaly）**：偏向特殊机制（全力值、温存、坚决、热意值、状态切换）

## 五、培育流程

### 5.1 培育准备

1. **选择育成偶像的P卡**：选择一张P卡进行培育
2. **查看初始属性**：查看该P卡的培育计划、初始三维、初始体力上限、专属技能卡

**不包含的系统**（相比学园）：
- ❌ 选择S卡作为培育支援（取消）
- ❌ 选择回忆卡（培育结束后才会生成回忆卡）
- ❌ 消耗AP（取消）

### 5.2 剧本选择

游戏共有多个不同的剧本，每个剧本有不同的回合数、难度、特殊规则。

**剧本列表**（改编自学园，使用闪耀色彩剧本名）：
1. **W.I.N.G.准优胜**（18回合）- 出道赛，难度：Regular/Pro/Master
2. **感谢祭**（20回合）- 粉丝回馈表演
3. **GRAD毕业典礼**（26回合）- 毕业演出
4. **Landing Point**（22回合）- 降落点
5. **STEP**（20回合）- 台阶
6. **Say Hello**（13回合）- 问候（新人友好）

### 5.3 回合制培育

在培育中，每回合以一周为单位，玩家可以选择以下行动之一：

#### 训练

是游戏最核心的培育玩法。玩家通过打出技能卡获取得分，以获取属性值和各种效果。

**训练流程**：
1. 选择训练类型（Vocal/Dance/Visual）
2. 进入打牌界面
3. 使用技能卡获取得分
4. 达到目标得分时获得Clear评价（可继续训练获得Perfect评价）
5. 训练结束后，根据完成程度给予属性值提升、P点数、P饮料和技能卡奖励

**训练评价**：
- **Perfect**：达到目标得分的2倍，获得最大奖励。如果提前结束，将多余的出牌回合数转换为2点体力回复
- **Clear**：达到目标得分，获得正常奖励
- **Fail**：未达到目标得分，无法获得饮料和技能卡奖励

**应援效果**：
- 训练过程中，可能会出现一些应援效果，需要在指定回合数内达成其要求
- 满足或未能满足应援效果的要求，会获得正面效果（如额外得分）或负面效果（如扣减体力、生成陷阱卡等）

**特殊训练模式**：
- **SP训练**：有时训练选项的左上角会出现SP标志，此时可以进行SP训练。SP训练有更高的目标得分要求，但能够提供更高的属性上升值
- **追加训练**：比赛的前一周固定为追加训练，有更高的目标得分和更多的出牌回合数。达到Clear评价以后的上升数值被称为追加Bonus，该部分的属性值会均分给全部三种属性
- **普通自主训练**：不需要玩家进行操作，消耗6点体力，对应属性值增加，获得20点P点数
- **SP自主训练**：根据角色SP课程概率触发，消耗8点体力，增加更多对应属性值，获得20点P点数

#### 自由活动

原"外出"系统改为"自由活动"，改成做一张地图，每次随机解锁三个地点，点击前往。

**地点类型**：
- 海滩：体力恢复+好感度
- 图书馆：全三维小幅增长
- 咖啡厅：好感度大幅增长
- 录音室：Vocal增长
- 舞蹈室：Dance增长
- 摄影棚：Visual增长
- 公园：体力恢复+随机事件
- 商业街：购买道具（未来扩展）
- 事务所：与其他偶像互动
- 演唱会场馆：特殊事件

**AI剧情生成**：
- 每次前往地点时，会触发一段AI生成的剧情
- 剧情内容会根据当前好感度等级、地点、回合数动态生成
- 剧情结束后，会获得对应的奖励（体力/属性/好感度）

#### 其他培育选项

- **上课**：通过消费体力，在三个不同的选项中选择一个，从而获得对应的属性值上升、技能卡等
- **活动补给**：向老师获取补给，固定获得一定量的P点数、一个P饮料和一张技能卡
- **相谈**：与老师相谈，消耗P点数购买技能卡或P饮料，以及强化/删除技能卡
- **休息**：体力不为满时可以选择。回复大量的体力后，直接进入下一周

### 5.4 比赛

**比赛时机**：
- 中间比赛（回合中期）
- 最终比赛（最后一回合）

**比赛流程**：
1. 进入打牌界面
2. 使用技能卡获取基础得分（技能卡效果包括：元气、干劲、专注、集中等属性，以及状态良好、好调等Buff）
3. 比赛的每个出牌回合都会被划分到一种属性主题（Vocal/Dance/Visual）
4. 在该回合中，技能卡获得的基础得分会乘以对应的**三维加成倍率**转换为最终得分
5. **三维加成倍率**：Vocal/Dance/Visual三维越高，在对应主题回合中获得的最终得分越高
6. 三种属性的回合占比以P卡的**审查基准**高低比例为准、顺序随机，但最后三回合的属性必定以审查基准高低决定顺序
7. 比赛结束后，根据总得分决定排名

**关键机制说明**：
- ✅ 技能卡效果：元气、干劲、专注、集中、活力、全力值等**属性**，以及状态良好、好调等**Buff**
- ✅ Vocal/Dance/Visual三维：作为**分数乘区**，在对应主题回合中提供得分倍率加成
- ✅ 审查基准：决定比赛中Vocal/Dance/Visual回合的分布比例
- ❌ 技能卡**不直接提升**Vocal/Dance/Visual三维

**应援效果**（仅最终比赛）：
- 最终比赛会出现一些应援效果，需要在指定回合数内达成其要求
- 达成或未能达成时会获得正面或负面效果

**排名与合格**：
- 达到前3名即算作比赛合格
- 比赛不合格则直接进入结局部分

**重试机制**：
- 每次培育有3次比赛重试机会（所有比赛共通）
- 除去每日第一次重试无消耗外，每次重试需要消耗一张"再挑战券"

### 5.5 培育完成与结局

**结局剧情（AI生成）**：
- 根据最终比赛的排名，会有对应的AI生成的结局剧情
- 剧情长度：200字左右
- 剧情内容：表现偶像的成长和对制作人的感激

**培育评价**：

培育评价主要由三个部分组成：
1. **最终比赛排名评价分**
   - 第1名：1700分
   - 第2名：900分
   - 第3名：500分
   - 第4-6名：0分

2. **最终比赛分数评价分**（根据得分X计算）
   - 0~5000：0.3X
   - 5001~10000：0.15X+750
   - 10001~20000：0.08X+1450
   - 20001~30000：0.04X+2250
   - 30001~40000：0.02X+2850
   - 40001~50000：0.01X+3250

3. **属性值评价分**
   - 最终比赛后三种属性值之和的2.3倍

上述三者的评价分分别向下取整后相加，即可得到培育评价分。

**培育评价等级**（学园偶像大师标准）：

| 培育评价分      | 培育评价 |
|---------------|---------|
| 18000+        | SS+     |
| 16000~17999   | SS      |
| 14500~15999   | S+      |
| 13000~14499   | S       |
| 11500~12999   | A+      |
| 10000~11499   | A       |
| 8000~9999     | B+      |
| 6000~7999     | B       |
| 4000~5999     | C+      |
| 3000~3999     | C       |
| 2000~2999     | D       |
| 1000~1999     | E       |
| 0~999         | F       |

**评价目标**：
- **初次通关目标**：C评价以上（3000分）
- **标准目标**：A评价以上（10000分）
- **高级目标**：S+评价以上（14500分）
- **终极目标**：SS+评价（18000分）

**奖励发放**：
- 根据培育评价等级，发放羽石、粉丝、经验值
- S评价示例：羽石2000、粉丝100000、经验值500

## 六、回忆卡系统

### 6.1 回忆卡生成

在培育完成后，会根据最终评价生成一张回忆卡。

**回忆卡内容**：
- **卡面**：使用该P卡的卡面（不再使用Live照片）
- **回忆卡名**：由AI生成一个专属的回忆卡名（如"星光闪耀的瞬间"）
- **培育能力**：根据最终评价随机生成（如"训练时Vocal额外+5"）
- **可继承技能卡**：根据最终评价从卡组中抽取3-6张技能卡，可在下次培育开始时或中期比赛后继承
- **继承时间点**：培育开始时（仅R技能卡）或中期比赛结束后

**不包含的系统**（相比学园）：
- ❌ 选择Live照片作为卡面（取消）
- ❌ 比赛技能卡组（取消）
- ❌ P卡和S卡专属技能卡无法作为可继承技能卡

### 6.2 回忆卡使用

回忆卡可以在下次培育时选择携带（最多4张），为P卡提供各项培育加成和可继承技能卡。

**回忆卡也可以从好友借用（未来扩展）**：
- 每次培育最多1张
- 每天最多3次（活动限时增加除外）

## 八、技术实现要点

### 8.1 前端实现

- **Vue 3 + TypeScript**：主要开发框架
- **卡牌UI**：Card.vue组件（支持拖拽、动画）
- **buff栏**：显示所有正面/负面效果及其层数/剩余回合数
- **打牌界面**：CardGame.vue组件（手牌、牌堆、弃牌堆、除外堆）
- **培育界面**：ProduceHost.vue组件（回合管理、行动选择、属性展示）
- **自由活动地图**：FreeActivityMap.vue组件（地点卡片、解锁动画）
- **剧情对话框**：GalDialogue.vue组件（角色立绘、对话气泡、打字机效果）

### 8.2 AI集成

- **提示词加工厂**：PromptFactory.ts（根据游戏状态动态组装提示词）
- **世界书区**：lore/文件夹（模块化存储角色人设、卡牌机制、好感度表现等）
- **iframe通信**：postMessage双向通信（Vue ↔ SillyTavern）
- **Card Generation Bot**：专用AI角色卡（生成专属技能卡）
- **剧情生成**：根据不同场景（训练、自由活动、结局）使用不同的提示词模板

### 8.3 数据管理

- **localStorage**：唯一的数据源（资源、抽卡数据、好感度、培育进度、回忆卡）
- **IndexedDB**：图片缓存（角色卡面、地点图片、buff图标）
- **JSON配置**：剧本配置、地点配置、P饮料配置
