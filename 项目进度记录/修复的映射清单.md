## ⚠️ 重要提示

这些映射是基于常见的日文罗马音规则推测的。**实际的文件名必须与你 GitHub 仓库中的文件名完全匹配！**

## 如何验证映射是否正确

### 1. 在浏览器中测试
```bash
# 打开构建后的文件
dist/偶像大师闪耀色彩/index.html
```

### 2. 打开图鉴页面
- 点击"偶像图鉴"按钮
- 启用开发工具："解锁全部角色"

### 3. 查看控制台错误
打开浏览器控制台（F12），查看是否还有"卡面加载失败"的错误。

### 4. 对比 GitHub 仓库文件名
访问你的 GitHub 仓库：
```
https://github.com/2426269/shinycolors-assets-cdn/tree/main/角色卡面
```

对于任何仍然失败的卡面：
1. 找到控制台中显示的卡名，如`【さかさ世界】芹沢あさひ`
2. 在 GitHub 中搜索包含`Serizawa.Asahi`的文件
3. 查看实际的文件名前缀（主题部分）
4. 更新`src/偶像大师闪耀色彩-gacha/data/name-mappings.ts`中的映射

### 5. 修正映射示例

如果实际文件名是 `Sakasa.Sekai_Serizawa.Asahi.webp`，则映射正确 ✅

如果实际文件名是 `Sakasama.Sekai_Serizawa.Asahi.webp`，则需要修改为：
```typescript
さかさ世界: 'Sakasama.Sekai',  // 修改为实际文件名
```

## 快速修复流程

1. 打开图鉴 → 解锁全角色
2. 记录所有"卡面加载失败"的错误
3. 在 GitHub 中查找对应角色的实际文件名
4. 更新 `name-mappings.ts`
5. 重新构建：`npm run build`
6. 刷新浏览器测试

```bash
npm run build
```
